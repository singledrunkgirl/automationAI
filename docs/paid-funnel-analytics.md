# Paid Funnel Analytics

Paid funnel events use stable snake_case event names. Most events include
`paid_funnel_event_version` so dashboards can filter or migrate safely.

## Event Contract

- `upgrade_cta_impressed`: an upgrade prompt became visible.
- `upgrade_cta_clicked`: a user clicked an upgrade prompt.
- `checkout_intent_clicked`: a plan checkout was requested from the client.
- `checkout_started`: a Stripe Checkout session or direct subscription-change
  checkout was started.
- `checkout_redirected`: the browser was redirected to Stripe Checkout.
- `checkout_succeeded`: Stripe Checkout completed or a direct paid plan change
  succeeded.
- `subscription_started`: Stripe confirmed a new paid subscription invoice.
- `subscription_changed`: Stripe confirmed a tier change on an existing
  subscription.
- `limit_hit`: a chat or agent request was blocked by a free or paid limit.
- `monthly_cap_hit`: legacy paid-only monthly cap event kept for existing
  dashboards.
- `add_credit_cta_impressed`: an add-credit prompt became visible.
- `add_credit_cta_clicked`: a user clicked an add-credit prompt.
- `add_credit_checkout_started`: an extra-usage credit checkout was created.
- `add_credit_checkout_succeeded`: an extra-usage credit checkout was paid.

## Join Keys

- `checkout_attempt_id`: generated by the app before checkout starts. Use this
  to join client intent, server checkout start, checkout success, and
  subscription lifecycle events.
- `stripe_checkout_session_id`: available after Stripe Checkout session
  creation. Use this for Stripe reconciliation and dedupe.
- `stripe_subscription_id`, `stripe_invoice_id`, and `stripe_customer_id`: use
  these for billing reconciliation.
- `userId` and `org_id`: use these for user/account funnel joins.

## Common Properties

- `surface`: UI location, such as `pricing_dialog`, `rate_limit_warning`, or
  `extra_usage_settings`.
- `source`: finer-grained trigger, such as `plan_card`, `limit_pressure`, or
  `buy_extra_usage_dialog`.
- `from_tier`, `to_tier`, `subscription_tier`: tier state around the event.
- `plan`, `billing_interval`, `billing_interval_count`, `quantity`: selected
  plan details.
- `checkout_type`: `new_subscription`, `subscription_change`,
  `extra_usage_purchase`, or `team_extra_usage_purchase`.
- `amount_dollars`, `checkout_amount_dollars`, `revenue_dollars`: money fields
  in USD unless `currency` says otherwise.
- `cap_reason`, `limit_type`, `reset_timestamp`: limit-pressure context.

## Data Minimization

- Do not include prompts, chat content, filenames, emails, or free-form user
  text in paid funnel events.
- Server-accepted `source` and `surface` labels are short allowlisted labels,
  not arbitrary client-provided strings.
- `checkout_attempt_id` and Stripe ids are for funnel stitching and billing
  reconciliation only.

## Suggested Readouts

- Signup to first usage: `user_signed_up -> hwai-usage_cost`.
- Free agent activation to paid:
  `user_signed_up -> hwai-free_agent_value_reached -> subscription_started`.
- Upgrade click to success: `upgrade_cta_clicked -> checkout_succeeded`,
  grouped by `surface`.
- Checkout start to subscription: `checkout_started -> subscription_started`,
  joined or filtered by `checkout_attempt_id`.
- Limit pressure to paid/add-credit: `limit_hit -> upgrade_cta_clicked` and
  `limit_hit -> add_credit_checkout_succeeded`.
- Referral revenue:
  `referred_signup_attributed -> referred_user_paid_conversion`, grouped by
  `referral_code`.
